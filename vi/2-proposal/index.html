AI AWS Architecture Reviewer

Nền tảng Serverless sử dụng AI để đánh giá sơ đồ kiến trúc AWS

1. Tóm tắt điều hành

AI AWS Architecture Reviewer là một nền tảng web serverless được thiết kế để hỗ trợ người dùng đánh giá sơ đồ kiến trúc AWS một cách tự động bằng AI và AWS Well-Architected Framework. Nền tảng cho phép người dùng upload sơ đồ kiến trúc thông qua ứng dụng web React, lưu trữ file upload an toàn trong Amazon S3, ghi metadata của mỗi lần review vào Amazon DynamoDB và kích hoạt một workflow review tự động.

Giải pháp sử dụng Amazon S3 để lưu trữ bản build production của frontend React, Amazon CloudFront để phân phối website đến người dùng và AWS WAF để bảo vệ CloudFront distribution khỏi các request có nguy cơ độc hại. Amazon API Gateway và AWS Lambda được sử dụng để xử lý upload, Amazon S3 Input Bucket dùng để lưu trữ sơ đồ kiến trúc được upload, Amazon EventBridge và AWS Step Functions dùng để khởi động và quản lý workflow review.

Trong workflow xử lý, AWS Lambda AI Analyzer đọc uploaded diagram từ Amazon S3 Input Bucket và chuẩn bị dữ liệu đầu vào để gửi sang Amazon Bedrock. Amazon Bedrock nhận diện các AWS services, connections, text notes và các thành phần kiến trúc từ sơ đồ, sau đó trả về architecture JSON có cấu trúc. Architecture JSON này được sử dụng làm dữ liệu trung tâm cho các bước phân tích tiếp theo.

Sau khi có architecture JSON, hệ thống gửi dữ liệu này sang AWS Lambda Cost Tool để ước tính chi phí hàng tháng dựa trên các AWS services được phát hiện trong sơ đồ. Lambda Cost Tool tạo cost estimation result, bao gồm danh sách dịch vụ, đơn vị tính chi phí, giả định sử dụng và chi phí ước tính theo tháng.

Cuối cùng, architecture JSON và cost estimation result được gửi lại sang Amazon Bedrock để thực hiện đánh giá tổng thể kiến trúc dựa trên AWS Well-Architected Framework. Amazon Bedrock phân tích các khía cạnh như Security, Reliability, Performance Efficiency, Cost Optimization, Operational Excellence và Sustainability, đồng thời giải thích chi phí và đề xuất hướng tối ưu.

Sau khi quá trình phân tích hoàn tất, AWS Lambda PDF Generator tạo báo cáo PDF bao gồm architecture review, cost estimation, cost explanation và optimization recommendations. Báo cáo được lưu vào Amazon S3 Report Bucket và lịch sử review được cập nhật trong Amazon DynamoDB. Amazon CloudWatch được sử dụng để ghi log, theo dõi metrics và tạo alarms cho các thành phần quan trọng. Amazon SNS được tích hợp với CloudWatch Alarm để gửi email cảnh báo khi hệ thống phát sinh lỗi. AWS IAM được dùng để áp dụng quyền truy cập theo nguyên tắc least privilege, trong khi AWS WAF bảo vệ frontend thông qua CloudFront distribution.

Mục tiêu của đề tài là cung cấp một giải pháp AWS serverless thực tế, có khả năng mở rộng và tối ưu chi phí, giúp sinh viên, người học và nhóm dự án cloud review kiến trúc AWS nhất quán hơn, giảm công sức review thủ công và cải thiện hiểu biết về các best practices trong thiết kế kiến trúc cloud.

2. Tuyên bố vấn đề

Vấn đề hiện tại

Việc review một kiến trúc AWS thủ công đòi hỏi kiến thức về nhiều dịch vụ AWS, các mẫu thiết kế cloud, best practices về bảo mật, chiến lược đảm bảo độ tin cậy, tối ưu hiệu năng, tối ưu chi phí và vận hành hệ thống. Đối với sinh viên, người mới học cloud hoặc các nhóm dự án, việc xác định một kiến trúc AWS có tuân thủ AWS Well-Architected Framework hay không thường gặp nhiều khó khăn.

Các thách thức phổ biến gồm:

  • Sơ đồ kiến trúc AWS thường được review thủ công và thiếu tính nhất quán.
  • Sinh viên có thể không biết kiến trúc của mình đang thiếu dịch vụ quan trọng nào hoặc có rủi ro thiết kế nào.
  • Review thủ công yêu cầu kinh nghiệm về cloud architecture và mất nhiều thời gian.
  • Chưa có một hệ thống tập trung để upload, review, lưu trữ và theo dõi lịch sử review kiến trúc.
  • Việc tạo một báo cáo review có cấu trúc rõ ràng bằng tay thường mất thời gian.
  • Các nhóm cần một giải pháp tự động có thể phân tích sơ đồ kiến trúc và đưa ra đề xuất cải thiện.

Giải pháp

AI AWS Architecture Reviewer cung cấp một workflow serverless tự động để review sơ đồ kiến trúc AWS. Người dùng truy cập ứng dụng web thông qua Amazon CloudFront được bảo vệ bởi AWS WAF. Frontend React được lưu trữ trong Amazon S3 frontend bucket và được phân phối đến người dùng thông qua CloudFront. AWS WAF được gắn với CloudFront distribution để theo dõi, phát hiện và giảm thiểu các request có nguy cơ độc hại trước khi chúng truy cập vào ứng dụng.

API Gateway chuyển upload request đến AWS Lambda Upload Service. Lambda function validate file được upload, lưu diagram vào Amazon S3 Input Bucket và ghi metadata ban đầu vào Amazon DynamoDB. Sau khi object được tạo trong input bucket, Amazon S3 phát sinh object-created event. Event này được route thông qua Amazon EventBridge để khởi động AWS Step Functions review workflow.

AWS Step Functions điều phối toàn bộ quá trình review. Đầu tiên, Step Functions gọi AWS Lambda AI Analyzer để đọc uploaded diagram từ S3 Input Bucket. Lambda AI Analyzer chuẩn bị dữ liệu đầu vào và gửi diagram sang Amazon Bedrock. Amazon Bedrock phân tích sơ đồ để nhận diện AWS services, connections, text notes và các thành phần kiến trúc quan trọng, sau đó trả về architecture JSON có cấu trúc.

Sau khi nhận được architecture JSON, Lambda AI Analyzer gửi dữ liệu này sang AWS Lambda Cost Tool. Lambda Cost Tool sử dụng danh sách AWS services trong architecture JSON để ước tính chi phí hàng tháng. Kết quả của bước này là cost estimation result, bao gồm các dịch vụ được phát hiện, giả định sử dụng, chi phí ước tính và các thành phần có khả năng ảnh hưởng lớn đến ngân sách.

Tiếp theo, architecture JSON và cost estimation result được gửi lại sang Amazon Bedrock để thực hiện đánh giá tổng thể kiến trúc. Ở bước này, Amazon Bedrock phân tích kiến trúc dựa trên AWS Well-Architected Framework, giải thích các rủi ro thiết kế, đánh giá chi phí, đề xuất cost optimization và đưa ra khuyến nghị cải thiện.

Sau khi Bedrock hoàn tất phần đánh giá tổng thể, AWS Lambda PDF Generator tạo báo cáo PDF từ kết quả AI review. Báo cáo được lưu trong Amazon S3 Report Bucket và review history được cập nhật trong Amazon DynamoDB. Người dùng có thể xem kết quả review và tải PDF report trực tiếp từ frontend.

Amazon CloudWatch được sử dụng để ghi logs, theo dõi metrics và tạo alarms cho Lambda, API Gateway, Step Functions, DynamoDB và EventBridge. Khi hệ thống phát sinh lỗi quan trọng, CloudWatch Alarm gửi cảnh báo đến Amazon SNS Topic, sau đó SNS gửi email alert đến các địa chỉ đã đăng ký và xác nhận subscription.

→ AWS WAF
→ Amazon CloudFront
→ S3 React Frontend
→ API Gateway
→ Lambda Upload Service
→ S3 Input Bucket
→ EventBridge
→ Step Functions
→ Lambda AI Analyzer
→ Amazon Bedrock sinh architecture JSON
→ Lambda Cost Tool ước tính monthly cost
→ Amazon Bedrock đánh giá tổng thể kiến trúc và tối ưu chi phí
→ Lambda PDF Generator
→ S3 Report Bucket
→ DynamoDB

Lợi ích và hoàn vốn đầu tư

Giải pháp cung cấp một công cụ thực tế để học tập và review thiết kế kiến trúc AWS. Hệ thống giúp giảm công sức review thủ công, cải thiện tính nhất quán trong đánh giá kiến trúc và hỗ trợ người dùng hiểu rõ hơn về AWS Well-Architected principles.

Các lợi ích chính gồm:

  • Tự động review kiến trúc AWS dựa trên Well-Architected best practices.
  • Cung cấp phản hồi nhanh hơn cho sinh viên và nhóm dự án.
  • Lưu trữ tập trung các sơ đồ đã upload và metadata review.
  • Theo dõi lịch sử review thông qua DynamoDB.
  • Tạo báo cáo PDF phục vụ tài liệu và thuyết trình.
  • Cho phép người dùng xem kết quả review và tải PDF report từ frontend.
  • Bổ sung lớp bảo vệ frontend bằng AWS WAF gắn với CloudFront distribution.
  • Cảnh báo lỗi vận hành thông qua CloudWatch Alarm và Amazon SNS.
  • Kiến trúc serverless giúp giảm công sức vận hành.
  • Thiết kế tối ưu chi phí vì hầu hết dịch vụ hoạt động theo mô hình pay-per-use.
  • Dễ mở rộng thông qua EventBridge, Step Functions, Lambda, S3 và DynamoDB.

Đề tài cũng tạo giá trị dài hạn như một nền tảng giáo dục có thể tái sử dụng cho việc học cloud architecture, review dự án AWS và phát triển các công cụ đánh giá cloud bằng AI trong tương lai.

3. Kiến trúc giải pháp

Nền tảng sử dụng kiến trúc AWS serverless event-driven. Frontend layer được host bằng Amazon S3, phân phối thông qua Amazon CloudFront và được bảo vệ bằng AWS WAF. API layer được xây dựng bằng Amazon API Gateway và AWS Lambda. File upload được lưu trong Amazon S3 Input Bucket, trong khi metadata review được lưu trong Amazon DynamoDB. Workflow review được kích hoạt bởi Amazon EventBridge và được điều phối bằng AWS Step Functions.

Trong Processing Layer, AWS Lambda AI Analyzer đọc uploaded diagram từ S3 Input Bucket và gửi diagram sang Amazon Bedrock để phân tích sơ đồ. Amazon Bedrock trả về architecture JSON, bao gồm các AWS services, connections, text notes và các thành phần kiến trúc được nhận diện. Sau đó, architecture JSON được gửi sang AWS Lambda Cost Tool để ước tính monthly cost. Kết quả cost estimation cùng với architecture JSON được gửi lại sang Amazon Bedrock để đánh giá tổng thể kiến trúc theo AWS Well-Architected Framework và tạo nội dung phân tích cuối cùng. AWS Lambda PDF Generator sử dụng kết quả này để tạo báo cáo PDF.

Kiến trúc bao gồm các bước chính sau:

  1. User truy cập web application thông qua Amazon CloudFront.
  2. AWS WAF được gắn với CloudFront distribution để kiểm tra request và bảo vệ frontend.
  3. CloudFront phân phối React frontend từ Amazon S3 frontend bucket.
  4. User submit architecture diagram thông qua frontend.
  5. API Gateway nhận upload request và gửi đến AWS Lambda Upload Service.
  6. Lambda Upload Service validate upload request và lưu uploaded diagram vào Amazon S3 Input Bucket.
  7. Amazon S3 phát sinh object-created event sau khi diagram được upload.
  8. Amazon EventBridge nhận object-created event và khởi động AWS Step Functions review workflow.
  9. AWS Step Functions điều phối review workflow và gọi AWS Lambda AI Analyzer.
  10. AWS Lambda AI Analyzer gửi uploaded diagram sang Amazon Bedrock để Bedrock nhận diện AWS services, connections, text notes và sinh architecture JSON.
  11. AWS Lambda AI Analyzer gửi architecture JSON sang AWS Lambda Cost Tool để ước tính monthly cost.
  12. AWS Lambda Cost Tool gửi cost estimation result cùng với architecture JSON sang Amazon Bedrock để đánh giá toàn bộ kiến trúc, giải thích chi phí và đề xuất tối ưu.
  13. AWS Lambda PDF Generator tạo PDF report từ kết quả final AI review.
  14. AWS Lambda PDF Generator lưu generated PDF report vào Amazon S3 Report Bucket.
  15. AWS Lambda PDF Generator cập nhật review history, report path và review status vào Amazon DynamoDB.
  16. Frontend hiển thị kết quả review và cho phép người dùng tải PDF report.
  17. Amazon CloudWatch ghi logs, theo dõi metrics và tạo alarms cho Lambda, Step Functions, API Gateway, DynamoDB và EventBridge.
  18. Amazon SNS nhận cảnh báo từ CloudWatch Alarm và gửi email alert khi hệ thống phát sinh lỗi quan trọng.
  19. AWS IAM áp dụng least privilege access giữa các services.

Các dịch vụ AWS sử dụng

  • AWS WAF: Bảo vệ CloudFront distribution khỏi các request có nguy cơ độc hại bằng các managed rule groups như Amazon IP Reputation List, Common Rule Set và Known Bad Inputs Rule Set.
  • Amazon CloudFront: Phân phối ứng dụng web React và cải thiện hiệu năng frontend.
  • Amazon S3 Frontend Bucket: Lưu React production build, bao gồm index.html và static assets.
  • Amazon API Gateway: Cung cấp API endpoints cho upload, review history, review detail và review status.
  • AWS Lambda Upload Service: Validate upload request, lưu uploaded diagrams, ghi metadata và xử lý các review data APIs.
  • Amazon S3 Input Bucket: Lưu trữ sơ đồ kiến trúc gốc do người dùng upload.
  • Amazon DynamoDB: Lưu review metadata, thông tin upload, review status và review history.
  • Amazon EventBridge: Bắt S3 object-created events và kích hoạt review workflow.
  • AWS Step Functions: Điều phối review workflow với retry và error handling.
  • AWS Lambda AI Analyzer: Đọc uploaded architecture diagram từ Amazon S3 Input Bucket, chuẩn bị dữ liệu đầu vào và gửi diagram sang Amazon Bedrock để nhận diện các AWS services, connections, text notes và sinh architecture JSON.
  • Amazon Bedrock: Được sử dụng trong hai giai đoạn chính. Giai đoạn đầu, Bedrock phân tích diagram để sinh architecture JSON có cấu trúc. Giai đoạn sau, Bedrock nhận architecture JSON và cost estimation result để đánh giá tổng thể kiến trúc theo AWS Well-Architected Framework, giải thích chi phí và đề xuất hướng tối ưu.
  • AWS Lambda Cost Tool: Nhận architecture JSON từ Lambda AI Analyzer, phân tích các AWS services được phát hiện trong sơ đồ và ước tính monthly cost. Kết quả cost estimation được gửi lại sang Amazon Bedrock để phục vụ bước đánh giá tổng thể và cost optimization.
  • AWS Lambda PDF Generator: Tạo PDF reports từ kết quả final AI review, bao gồm architecture review, cost estimation, cost explanation và optimization recommendations.
  • Amazon S3 Report Bucket: Lưu generated PDF reports.
  • Amazon CloudWatch: Cung cấp logging, monitoring, metric filters, alarms và troubleshooting cho Lambda, API Gateway, Step Functions, DynamoDB và EventBridge.
  • Amazon SNS: Nhận alert từ CloudWatch Alarm và gửi email cảnh báo khi hệ thống phát sinh lỗi quan trọng.
  • AWS IAM: Kiểm soát quyền truy cập của các services bằng least privilege access.

Thiết kế thành phần

  • Frontend and Protection Layer: Ứng dụng React được host trong Amazon S3, phân phối thông qua Amazon CloudFront và được bảo vệ bằng AWS WAF. Frontend gồm các trang Dashboard, Upload Diagram, Review Progress, Review History, Report Detail và Settings.
  • API Layer: Amazon API Gateway cung cấp các endpoints như POST /upload, GET /reviews, GET /reviews/{reviewId} và GET /reviews/{reviewId}/status.
  • Upload Processing Layer: AWS Lambda Upload Service validate uploaded files, tạo unique review ID, lưu file vào S3 và ghi metadata vào DynamoDB.
  • Storage Layer: Amazon S3 Input Bucket lưu uploaded diagrams, Amazon S3 Report Bucket lưu generated PDF reports, DynamoDB lưu metadata và review history.
  • Workflow Layer: EventBridge nhận object-created events từ S3 và khởi động Step Functions để điều phối quá trình review.
  • AI Processing Layer: AWS Lambda AI Analyzer đọc uploaded diagram từ S3 Input Bucket và gửi diagram sang Amazon Bedrock để trích xuất thông tin kiến trúc. Amazon Bedrock nhận diện AWS services, connections, text notes và trả về architecture JSON. Architecture JSON sau đó được dùng làm dữ liệu đầu vào cho bước ước tính chi phí và bước đánh giá tổng thể kiến trúc.
  • Cost Analysis Component: AWS Lambda Cost Tool nhận architecture JSON, xác định các AWS services xuất hiện trong sơ đồ và ước tính monthly cost dựa trên các giả định sử dụng ở mức demo. Kết quả cost estimation giúp báo cáo không chỉ đánh giá thiết kế kiến trúc mà còn cung cấp góc nhìn về chi phí vận hành dự kiến.
  • Final AI Review Component: Amazon Bedrock nhận architecture JSON và cost estimation result để đánh giá toàn bộ kiến trúc theo AWS Well-Architected Framework. Kết quả phân tích bao gồm điểm mạnh, rủi ro, vấn đề cần cải thiện, giải thích chi phí và đề xuất tối ưu.
  • Reporting Layer: AWS Lambda PDF Generator nhận kết quả final AI review từ Amazon Bedrock, tạo PDF report và lưu báo cáo vào Amazon S3 Report Bucket.
  • Monitoring and Alerting Layer: Amazon CloudWatch ghi logs, theo dõi metrics, tạo metric filters và CloudWatch Alarms. Amazon SNS nhận cảnh báo từ CloudWatch Alarm và gửi email alert đến địa chỉ đã subscribe.
  • Security Layer: AWS IAM áp dụng least privilege access giữa các services. AWS WAF bảo vệ frontend bằng cách kiểm tra request trước CloudFront và giảm thiểu các request có nguy cơ độc hại.

4. Triển khai kỹ thuật

Các giai đoạn triển khai

Dự án được triển khai theo nhiều giai đoạn, bắt đầu từ phát triển frontend và chức năng upload cơ bản, sau đó mở rộng sang event-driven AI workflow, PDF reporting, monitoring, alerting và security.

  • Phát triển frontend và hosting: Xây dựng frontend React bằng Vite, tạo các trang chính, build production files, upload lên Amazon S3, phân phối frontend thông qua Amazon CloudFront và gắn AWS WAF với CloudFront distribution để tăng cường bảo vệ frontend.
  • Triển khai upload backend: Tạo S3 Input Bucket, DynamoDB review table, Lambda execution role và Lambda Upload Service để xử lý file upload và lưu metadata.
  • API Gateway và Review Data API: Tạo các API Gateway routes cho upload diagram và lấy dữ liệu review. Cấu hình CORS để cho phép request từ CloudFront frontend đã deploy.
  • Event-Driven Review Workflow: Cấu hình S3 object-created events, route event qua EventBridge và khởi động Step Functions workflow để xử lý uploaded architecture diagram.
  • AI Analysis và Reporting: Sử dụng Lambda AI Analyzer để đọc uploaded diagram và gửi diagram sang Amazon Bedrock nhằm sinh architecture JSON. Sau đó, architecture JSON được gửi sang Lambda Cost Tool để ước tính monthly cost. Kết quả architecture JSON và cost estimation result được gửi lại sang Amazon Bedrock để đánh giá tổng thể kiến trúc theo AWS Well-Architected Framework. Cuối cùng, Lambda PDF Generator tạo PDF report và lưu vào S3 Report Bucket.
  • Monitoring, Alerting và Security: Giám sát logs, metrics và workflow execution bằng CloudWatch. CloudWatch Alarm gửi cảnh báo đến Amazon SNS Topic để gửi email alert khi hệ thống phát sinh lỗi. AWS IAM áp dụng least privilege permissions, trong khi AWS WAF bảo vệ frontend thông qua CloudFront distribution.

Yêu cầu kỹ thuật

  • React frontend sử dụng Vite.
  • Amazon S3 frontend bucket cho React production build.
  • Amazon CloudFront distribution với Origin Access Control.
  • AWS WAF Web ACL hoặc protection pack gắn với CloudFront distribution.
  • API Gateway cho backend APIs.
  • AWS Lambda cho upload handling và review API logic.
  • Amazon S3 Input Bucket cho uploaded diagrams.
  • Amazon DynamoDB cho metadata và review history.
  • Amazon EventBridge cho event-driven workflow triggering.
  • AWS Step Functions cho review orchestration.
  • AWS Lambda AI Analyzer cho diagram parsing.
  • Amazon Bedrock cho diagram-to-JSON extraction và final AI architecture review.
  • AWS Lambda Cost Tool cho monthly cost estimation dựa trên architecture JSON.
  • AWS Lambda PDF Generator cho report generation.
  • Amazon S3 Report Bucket cho PDF storage.
  • Amazon CloudWatch cho logging, monitoring, metric filters và alarms.
  • Amazon SNS cho email alert khi CloudWatch Alarm phát hiện lỗi quan trọng.
  • IAM cho least privilege access control.

6. Ước tính ngân sách

Dự án sử dụng các dịch vụ AWS serverless nên chi phí chủ yếu phụ thuộc vào mức sử dụng. Trong giai đoạn MVP và demo vừa, lượng truy cập dự kiến không quá lớn nhưng hệ thống vẫn có đầy đủ các thành phần chính như frontend hosting, frontend protection, API backend, file storage, metadata database, event-driven workflow, AI analysis, PDF report generation, monitoring và alerting.

Giả định tính toán

Ước tính ngân sách được tính dựa trên quy mô demo vừa như sau:

  • Số lượng người dùng: khoảng 5–10 users.
  • Số lượng diagram upload: khoảng 200–300 diagrams/tháng.
  • Dung lượng trung bình mỗi diagram: khoảng 3–5 MB.
  • Số lượng PDF report tạo ra: khoảng 200–300 reports/tháng.
  • Số lần chạy review workflow: khoảng 200–300 executions/tháng.
  • Số API requests: khoảng 5.000–15.000 requests/tháng.
  • Mỗi workflow gồm các bước: upload diagram, lưu metadata, kích hoạt workflow, gửi diagram sang Bedrock để sinh architecture JSON, ước tính monthly cost bằng Lambda Cost Tool, gửi architecture JSON và cost estimation result sang Bedrock để đánh giá tổng thể, tạo PDF report, lưu report và cập nhật review history.
  • Amazon Bedrock được sử dụng để phân tích kiến trúc ở mức demo, không phải production traffic lớn.
  • AWS WAF được sử dụng để bảo vệ CloudFront distribution ở mức cơ bản với managed rule groups.

Chi phí hạ tầng ước tính

  • Amazon S3: khoảng 0.10–0.50 USD/tháng
    Dùng để lưu React frontend build files, uploaded diagrams và generated PDF reports. Với dung lượng vài GB trong giai đoạn demo, chi phí S3 rất thấp.

  • Amazon CloudFront: khoảng 0.00–1.00 USD/tháng
    Dùng để phân phối frontend web application. Với lượng truy cập demo vừa, chi phí CloudFront thường thấp vì dữ liệu truyền tải chưa nhiều.

  • AWS WAF: phát sinh chi phí tùy theo số lượng Web ACL, rule groups và số lượng request được kiểm tra.
    Trong dự án này, AWS WAF được sử dụng ở mức demo để bảo vệ CloudFront distribution bằng các managed rule groups. Chi phí thực tế phụ thuộc vào cấu hình Web ACL và lượng request truy cập frontend.

  • Amazon API Gateway: khoảng 0.05–0.20 USD/tháng
    Dùng để xử lý các API requests như POST /upload, GET /reviews, GET /reviews/{reviewId} và GET /reviews/{reviewId}/status.

  • AWS Lambda: khoảng 0.00–0.70 USD/tháng
    Dùng để xử lý upload, validate file, ghi metadata, đọc uploaded diagram, điều phối dữ liệu architecture JSON, ước tính monthly cost thông qua Lambda Cost Tool và tạo PDF reports. Với số lượng request ở mức demo, chi phí Lambda vẫn rất thấp.

  • AWS Lambda Cost Tool: khoảng 0.00–0.20 USD/tháng
    Dùng để nhận architecture JSON, xác định các AWS services trong sơ đồ và ước tính monthly cost. Với quy mô demo 200–300 diagrams/tháng, chi phí chạy Lambda Cost Tool rất thấp. Chi phí thực tế phụ thuộc vào thời gian chạy Lambda, số lượng services cần tính toán và độ phức tạp của logic cost estimation.

  • Amazon DynamoDB: khoảng 0.05–0.30 USD/tháng
    Dùng để lưu review metadata, review history và review status. Dữ liệu mỗi review nhỏ nên chi phí DynamoDB thấp trong giai đoạn demo.

  • Amazon EventBridge: khoảng 0.01–0.05 USD/tháng
    Dùng để route S3 object-created events và kích hoạt workflow sau khi user upload diagram.

  • AWS Step Functions: khoảng 0.00–0.30 USD/tháng
    Dùng để điều phối review workflow. Với khoảng 200–300 workflow executions/tháng, chi phí Step Functions vẫn rất thấp nếu số state transitions không quá lớn.

  • Amazon SNS: khoảng 0.00–0.10 USD/tháng
    Dùng để gửi email alert khi CloudWatch Alarm phát hiện lỗi quan trọng trong hệ thống.

  • Amazon CloudWatch: khoảng 0.50–2.00 USD/tháng
    Dùng để lưu logs, theo dõi metrics, tạo metric filters, tạo alarms và hỗ trợ troubleshooting cho Lambda, API Gateway, Step Functions, DynamoDB và EventBridge. Chi phí CloudWatch phụ thuộc vào lượng logs được ghi ra trong quá trình test và debug.

  • Amazon Bedrock: khoảng 5.00–35.00 USD/tháng
    Dùng trong hai giai đoạn: phân tích diagram để sinh architecture JSON và đánh giá tổng thể kiến trúc dựa trên architecture JSON cùng cost estimation result. Đây là thành phần có chi phí biến động lớn nhất vì phụ thuộc vào model được chọn, số lượng input/output tokens, kích thước diagram, độ dài architecture JSON, độ chi tiết của cost estimation result và số lần gọi AI. Nếu sử dụng model nhỏ, giới hạn độ dài prompt và rút gọn dữ liệu đầu vào, chi phí sẽ thấp hơn.

7. Đánh giá rủi ro

Ma trận rủi ro

  • File diagram không hợp lệ hoặc không được hỗ trợ: Ảnh hưởng trung bình, xác suất trung bình.
  • Kết quả AI review không chính xác hoặc chưa đầy đủ: Ảnh hưởng cao, xác suất trung bình.
  • Workflow thất bại trong quá trình extraction hoặc report generation: Ảnh hưởng trung bình, xác suất trung bình.
  • Lỗi CORS giữa CloudFront và API Gateway: Ảnh hưởng trung bình, xác suất trung bình.
  • CloudFront cache vẫn phục vụ frontend build cũ: Ảnh hưởng thấp đến trung bình, xác suất trung bình.
  • AWS WAF rule chặn nhầm request hợp lệ nếu cấu hình rule quá nghiêm ngặt: Ảnh hưởng trung bình, xác suất thấp đến trung bình.
  • IAM permissions quá rộng hoặc quá hạn chế: Ảnh hưởng cao, xác suất trung bình.
  • Chi phí Amazon Bedrock tăng do token usage cao: Ảnh hưởng trung bình, xác suất thấp.
  • Bedrock nhận diện sai AWS services hoặc connections từ diagram: Ảnh hưởng cao, xác suất trung bình.
  • Architecture JSON sinh ra chưa đầy đủ hoặc không đúng cấu trúc: Ảnh hưởng cao, xác suất trung bình.
  • Cost estimation không chính xác do thiếu usage assumptions: Ảnh hưởng trung bình, xác suất trung bình.
  • Chi phí Amazon Bedrock tăng do gọi AI ở cả bước sinh JSON và bước final review: Ảnh hưởng trung bình, xác suất trung bình.
  • CloudWatch Metric Filter bắt nhầm từ khóa log bình thường và tạo alarm giả: Ảnh hưởng thấp đến trung bình, xác suất trung bình.

Chiến lược giảm thiểu

  • Validate file type và file size trước khi lưu upload.
  • Sử dụng structured prompts cho Amazon Bedrock.
  • Sử dụng Step Functions retry và error handling.
  • Lưu workflow status trong DynamoDB.
  • Cấu hình CORS chính xác cho localhost và CloudFront domain.
  • Tạo CloudFront invalidation sau mỗi lần deploy frontend.
  • Cấu hình AWS WAF managed rule groups ở Count mode trước khi chuyển sang Block mode.
  • Theo dõi AWS WAF sampled requests để kiểm tra request hợp lệ có bị rule match hay không.
  • Áp dụng least privilege IAM policies.
  • Theo dõi Lambda, API Gateway, Step Functions, Bedrock và WAF usage thông qua CloudWatch.
  • Sử dụng AWS Budgets để theo dõi chi phí hàng tháng dự kiến.
  • Chuẩn hóa prompt cho bước diagram-to-JSON extraction.
  • Validate architecture JSON trước khi gửi sang Lambda Cost Tool.
  • Sử dụng default usage assumptions rõ ràng cho cost estimation.
  • Giới hạn kích thước diagram và độ dài dữ liệu gửi sang Bedrock.
  • Tách riêng prompt sinh architecture JSON và prompt final architecture review để dễ kiểm soát output.
  • Tinh chỉnh CloudWatch Metric Filter để chỉ bắt lỗi thật như ERROR, Exception, Traceback, AccessDenied, ValidationException, Timeout thay vì các từ khóa quá rộng.

Kế hoạch dự phòng

  • Nếu AI analysis thất bại, đánh dấu review status là failed trong DynamoDB.
  • Nếu PDF generation thất bại, giữ lại AI review result và retry quá trình tạo PDF.
  • Nếu CloudWatch hoặc SNS alert chưa hoạt động, kiểm tra alarm action, SNS subscription và trạng thái confirmed của email.
  • Nếu AWS WAF chặn nhầm request hợp lệ, chuyển rule sang Count mode, kiểm tra sampled requests và điều chỉnh rule action.
  • Nếu CloudFront vẫn phục vụ file cũ, tạo invalidation và redeploy frontend.
  • Nếu Bedrock usage trở nên tốn kém, giới hạn input size hoặc giảm số lượng review requests.
